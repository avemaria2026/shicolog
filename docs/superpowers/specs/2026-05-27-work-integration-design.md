# シコログ：作品取り込み機能 設計書

**日付**: 2026-05-27
**ステータス**: ブレインストーミング完了、実装プラン未作成

## 1. 概要

シコログの記録に「作品（FANZA動画タイトル＋CID＋サムネ画像）」フィールドを追加し、
記録時にDMM Web Service APIから女優の作品一覧を取得して、ユーザーがアプリ内で
選択できるようにする。

データ構造：`{誰で, 作品, ジャンル}` の3軸記録に拡張。

## 2. 解決する課題

1. **記録の詳細化**：「何を見たか」が女優名だけでなく具体的作品まで残せる
2. **Xシェアのカード表示強化**：FANZA作品個別ページはOG画像（パッケージ）を持っているため、
   シェアしたとき動画のサムネカードが確実に表示される（現状はカードが出ない or シコログのカード）
3. **長期のストック化**：「今月よく見た作品TOP3」「年間シコリスト」みたいな集計に育てる土台

## 3. 非ゴール

- 作品の自動 enrichment（既存記録への遡及付与）はやらない
- 視聴履歴の追跡や FANZA アカウント連携はやらない
- DMM以外のサービス（DUGA等）連携はやらない

## 4. アーキテクチャ

```
[ ブラウザ：shicolog ]
        ↓ fetch('/api/fanza-search?actress=松本いちか')
[ Vercel Serverless Function: /api/fanza-search.js ]  ← 新規追加
        ↓ DMM Web Service API 呼び出し（サーバー側、APIキー安全）
[ DMM API (api.dmm.com/affiliate/v3/ItemList) ]
        ↓ JSON返却
[ 関数が整形・フィルタしてフロントへ返却 ]
        ↓
[ ブラウザ：モーダル内に作品カードリスト表示 ]
```

### 構成変更点

- shicolog はこれまで100%静的サイトだったが、**Vercel Serverless Functions** を1つ追加する
- 起動時の外部送信はゼロのまま（API呼び出しは「作品を探す」ボタンをユーザーがタップした時のみ）
- API キーは Vercel 環境変数で管理、クライアントには露出させない
  - `DMM_API_ID` — DMM Web Service の API ID（要：DMM管理画面から取得）
  - `DMM_AFFILIATE_ID` — `vocchan-015`（取得済み）

### 新規ファイル

- `api/fanza-search.js` — Vercel Serverless Function
- `api/_mockProducts.js` — フェーズA用のモックデータ（フェーズB完了後に削除）
- `vercel.json` — 必要なら（多くの場合自動検出で不要）

## 5. データ構造

### 現行レコード

```json
{
  "id": "1748334000000-abc123",
  "datetime": "2026-05-27T12:00:00.000Z",
  "who": "松本いちか",
  "how": "潮吹き"
}
```

### 拡張後レコード

```json
{
  "id": "1748334000000-abc123",
  "datetime": "2026-05-27T12:00:00.000Z",
  "who": "松本いちか",
  "work": {
    "cid": "ssis-456",
    "title": "「即イキ即イラマ」松本いちか",
    "imageUrl": "https://pics.dmm.co.jp/.../ssis456pl.jpg",
    "url": "https://www.dmm.co.jp/digital/videoa/-/detail/=/cid=ssis-456/"
  },
  "how": "潮吹き"
}
```

### フィールド仕様

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `work` | object\|undefined | × | 作品情報。未設定なら従来通り女優+ジャンルのみ |
| `work.cid` | string | ○ | FANZA作品ID（例：`ssis-456`） |
| `work.title` | string | ○ | 作品タイトル |
| `work.imageUrl` | string | ○ | パッケージ画像URL（履歴サムネ表示用） |
| `work.url` | string | ○ | FANZA作品ページURL（タップ・シェアで使用） |

### 後方互換

- 既存レコードは `work` フィールド無しのままで動作
- `work` を読む箇所はすべて `if (record.work)` でガード
- マイグレーションスクリプトは実装しない（必要ない）

### エクスポート/インポート

- **CSV**：列追加 `work_cid`, `work_title`（imageUrl/urlは派生可能なので省略）
- **JSON書き出し**：`work` オブジェクト全体を含める
- **引き継ぎコード**：JSON互換のためそのまま動作

## 6. UX フロー

### 記録モーダル（新規作成時）

```
┌─ 記録する ─────────────────┐
│ 誰？                       │
│ [松本いちか....] [🎲] [🔎] │
│ [☆松本いちか] [☆河北彩花] │ ← 既存の候補チップ
│                            │
│ 作品？                     │ ← NEW セクション
│ [📥 この女優の作品を探す]   │ ← タップで下に展開
│ ┌──────────────────────┐   │
│ │ 🎬 [サムネ] 作品A     │   │
│ │ 🎬 [サムネ] 作品B     │   │
│ │ 🎬 [サムネ] 作品C     │   │
│ │ ⋯（最大8件）         │   │
│ └──────────────────────┘   │
│ ※選択するとサムネ＋タイトル │
│   表示・「解除」リンク付き  │
│                            │
│ ジャンルは？               │
│ [グラビア][グラビアアイドル][アイドル][知り合い] │
│ [潮吹き....] [🎲] [🔎]     │
│                            │
│ [スキップ] [保存]          │
└────────────────────────────┘
```

### インタラクション仕様

1. **作品探すボタン**：押下時にAPI呼び出し → ローディング表示 → 結果（最大8件）を縦リスト表示
2. **作品カード**：横長レイアウト（サムネ60×60 + タイトル + CID）。タップで選択状態に
3. **選択後**：作品リスト折りたたみ → 選択した作品が大きめのカードで表示（サムネ100×100 + タイトル + CID + 解除リンク）
4. **未選択時の保存**：`work` フィールド無しで保存（現行と同じ動作）
5. **再検索**：選択を解除すると「📥 この女優の作品を探す」ボタンに戻る
6. **エラー時**：「作品が見つかりませんでした」「通信エラー」等のメッセージ表示

### 編集モーダル

- 既存記録（`work` 無し）：上記の通常フローと同じ。新規で `work` を追加可能
- `work` 付き記録：保存済みの作品カードが表示されている。解除すれば検索フローへ

## 9. Xシェアの強化

### 作品ありレコード

```
松本いちかさんの「即イキ即イラマ」で賢者になりました
— シコログ📓 https://shicolog.vercel.app/
https://www.dmm.co.jp/digital/videoa/-/detail/=/cid=ssis-456/   ← 個別作品URL
#シコログ #松本いちか
```

- FANZA個別作品URLは `og:image`（パッケージ画像）を持っているため、
  Twitterカードに作品サムネが表示される可能性が高い
- アフィリラッパー（`al.dmm.co.jp`）経由でアフィリID付与

### 作品なしレコード

現行通り。女優FANZA検索URL or ジャンル検索URL。

## 10. DMM Web Service API 仕様

### エンドポイント

```
GET https://api.dmm.com/affiliate/v3/ItemList
```

### 主要パラメータ

| パラメータ | 値 |
|---|---|
| `api_id` | 環境変数 `DMM_API_ID` |
| `affiliate_id` | 環境変数 `DMM_AFFILIATE_ID` (`vocchan-015`) |
| `site` | `FANZA` |
| `service` | `digital` |
| `floor` | `videoa`（一般動画） |
| `article` | `actress` |
| `article_id` | 女優ID ← **要：女優名→ID変換** |
| `hits` | `8`（取得件数） |
| `sort` | `date`（新着順）or `rank`（人気順） |
| `output` | `json` |

### 女優ID取得の補助 API

DMM API では女優名→女優IDの変換が必要。`ActressSearch` API を使用：

```
GET https://api.dmm.com/affiliate/v3/ActressSearch?keyword=松本いちか&output=json
```

または、**`keyword` パラメータで作品検索する形にして女優ID変換を省略する**シンプル化も検討可能：

```
GET .../ItemList?keyword=松本いちか&...
```

ただし keyword 検索だと「女優名」以外（タイトルに含まれる場合等）もヒットするのでノイズが入る。
**設計判断**：まず `keyword` 方式で実装、ノイズが目立つようなら `ActressSearch` 二段呼び出しに切り替える。

### `/api/fanza-search.js` の責務

1. クエリパラメータ `actress` を受け取る
2. DMM API に keyword 検索（フェーズBで実装）/ モックデータから検索（フェーズA）
3. レスポンスを以下の形に整形：
   ```json
   [
     { "cid": "ssis-456", "title": "...", "imageUrl": "...", "url": "..." },
     ...（最大8件）
   ]
   ```
4. CORS ヘッダー：同一オリジンなので不要（shicolog.vercel.app が両方ホスト）
5. キャッシュ：とりあえず無し。問題出たら Vercel Edge Cache 検討
6. エラーハンドリング：API エラー時は HTTP 502 + エラーJSON

## 11. 実装フェーズ

### フェーズA：API ID 待ちでも進められる作業

1. `api/_mockProducts.js` 作成：松本いちか・美谷朱里など主要女優の作品サンプルを手書き（5件ずつ）
2. `api/fanza-search.js` 作成（モック版）：`actress` クエリ受けて `_mockProducts.js` から該当女優の作品を返す
3. モーダルHTML：「作品を探す」ボタン＋作品リスト用 div 追加
4. JS：作品検索→表示→選択→保存のロジック
5. データ層：`addRecord` / `updateRecord` で `work` を受け付ける
6. 履歴描画：`work` があればサムネ＋タイトル表示
7. 統計描画：作品ランキング集計&表示
8. Xシェア：`shareRecord` で `work.url` を使う
9. CSV/JSON エクスポート・インポート対応
10. ローカル動作確認 → コミット → push（モック状態で本番に上がる）

### フェーズB：API ID 取得後

1. Vercelダッシュボードで環境変数 `DMM_API_ID`, `DMM_AFFILIATE_ID` 設定
2. `api/fanza-search.js` のモック部分を実 API 呼び出しに差し替え
3. `_mockProducts.js` 削除
4. デプロイ → 本番でテスト
5. 期待通り動いたらコミット → push

## 12. テスト方針

- ユニットテスト：シコログには既存のテストフレームが無いため、手動テストでカバー
- 手動テスト項目（フェーズA完了時）：
  - 新規記録：作品選択ありで保存→履歴に表示される
  - 新規記録：作品選択なしで保存→従来通り
  - 編集：作品の追加・変更・解除
  - 統計：作品ランキングが正しく集計される
  - X シェア：作品ありの場合、作品URLが含まれる
  - CSV書き出し：work_cid, work_title 列が出る
  - JSON 書き出し/取り込み：`work` がラウンドトリップする
  - 引き継ぎコード：`work` 付きで他端末に移行可能
- 手動テスト項目（フェーズB完了時）：
  - 実 API で女優検索 → 作品が返ってくる
  - エラー時のメッセージ表示
  - キャッシュなしでも体感速度が許容範囲

## 13. リスク・懸念点

| リスク | 対応 |
|---|---|
| DMM API が CORS を許可してくれて、ブラウザ直叩きできる場合 | 確認後、シンプルなアーキにできる（serverless 関数不要）。ただしAPIキー露出が代償 |
| API レート制限超過 | 個人利用前提なので問題ない見込み。問題出たら sessionStorage キャッシュ追加 |
| 女優名→女優ID 変換の失敗（DMMにいない・別名義） | keyword 検索でフォールバック、エラー時は「見つかりません」表示 |
| FANZA 個別作品URLが Twitter カードを返してくれない | フェーズB完了後に1回テスト投稿して確認。ダメなら撤退判断（カードは諦めて affiliate URL だけ仕込む） |
| 既存ユーザーのキャッシュ問題 | SW バージョン bump で対応 |
| 「ジャンル」が手入力のままだと作品ランキングと整合性取りにくい | フェーズC案：作品が選ばれた時に「この作品のジャンル候補」を自動サジェストするのは将来検討 |
